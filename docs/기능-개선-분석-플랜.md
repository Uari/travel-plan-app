## 1. 🏠 홈 — 여행 요약 대시보드로 재편

### 목표
"다트 랜덤 여행지 게임"이 홈을 차지하던 구조를, **여행 요약이 먼저 보이는 대시보드**로 바꾼다.

### 구현 내용
1. **요약 카드 섹션(신규, 홈 최상단)**
   - D-day(시작일 기준), 여행 이름/기간
   - 확정 숙소(있으면 이름, 없으면 "미정")
   - 다음(가장 가까운) 일정 1건
   - 총비용 / 1인당
   - 준비물 진행률(%)
   - 멤버 수
   - 각 카드 탭 → 해당 탭으로 이동(숙소/일정/비용/준비물)
2. **다트 게임 분리**
   - 홈에서 상시 노출 대신 "🎯 여행지 정하기" 섹션/버튼으로 이동(접힌 형태 또는 별도 시트)
   - 기존 다트 인터랙션/컨페티는 그대로 재사용
3. **제외 지역 localStorage → DB 이전** 🔴
   - 현재 `localStorage('travelplan_excluded')` → `excluded_regions` 테이블 사용(스키마엔 이미 설계됨)
   - 트립 단위로 저장되어 **팀원 전원이 공유**
   - 마이그레이션: 테이블 존재 확인 후 없으면 생성

### 기술 / DB
- 요약 데이터: 기존 `trips`, `plans`, `expenses`, `accommodations`, `checklist` 쿼리 조합(대부분 Context/훅 재사용)
- DB: `excluded_regions (id, trip_id, region_name, created_by, created_at)` 사용/생성

### 영향 파일
- `src/pages/DashboardPage.jsx` (대폭 개편), `DashboardPage.css`
- 제외지역 로직: `DashboardPage.jsx` (localStorage → supabase)

## 2. 🏨 숙소 — 정렬 옵션

### 목표
숙소 후보 목록을 **좋아요순 / 가격 낮은순 / 최신순**으로 전환할 수 있게 한다.

### 구현 내용
- 목록 상단에 세그먼트 토글: `추천순(좋아요) · 가격↓ · 최신순`
- 정렬 로직은 기존 `useMemo` 확장:
  - 좋아요순: 현재 로직(확정 숙소 최상단 유지)
  - 가격 낮은순: `price` 오름차순(0/미입력은 뒤로)
  - 최신순: `created_at` 내림차순
- 확정 숙소는 어떤 정렬에서도 최상단 유지(현행 유지)
- 정렬 바뀔 때 기존 `layout` 애니메이션으로 부드럽게 재배치

### 영향 파일
- `src/pages/AccommodationPage.jsx`, `AccommodationPage.css`

### 난이도: 하 (반나절)

## 3. 🗓️ 일정 — 드래그 정렬 + 지도 연동

### 3-1. 드래그 앤 드롭 정렬
- Day 그룹 내에서 일정 카드를 **드래그로 순서 변경**
- 저장: `plans`에 정렬용 컬럼 `sort_order`(정수) 추가 → 드래그 후 순서 일괄 업데이트
- 라이브러리: 이미 있는 **framer-motion의 `Reorder`** 컴포넌트 사용(추가 의존성 없음)
- 현재 `time_label` 자동정렬 → "수동 정렬 우선, 없으면 시간순" 규칙으로 변경

### 3-2. 지도 연동 (핀 + 하루 동선)
- 일정의 `location`(장소)을 좌표로 → 지도에 핀 표시, 하루 일정 순서대로 선으로 동선 표시
- 필요 데이터: `plans`에 `lat`, `lng` 컬럼 추가(장소 검색/선택 시 저장)
- **지도 라이브러리 결정 필요** (아래 8번)

### 기술 / DB
- DB: `plans` 테이블에 `sort_order INT`, `lat FLOAT8`, `lng FLOAT8` 컬럼 추가(마이그레이션)
- 드래그: framer-motion `Reorder.Group` / `Reorder.Item`
- 지도: 8번에서 택1

### 영향 파일
- `src/pages/PlanPage.jsx`, `PlanPage.css`, 지도용 신규 컴포넌트

### 난이도: 드래그 중 / 지도 중~상 (지도 라이브러리 선택에 따라)

## 6. 🛖 로비 — 초대 공유 링크 + QR

### 목표
초대 코드 복사 외에, **공유 링크와 QR 코드**로 더 쉽게 초대.

### 구현 내용
- 트립 상단/로비에 "초대하기" 시트:
  - **공유 링크**: `https://<앱주소>/join/<코드>` 형태(입장 화면으로 딥링크)
  - **QR 코드**: 위 링크를 QR로 생성(옆 사람이 스캔해서 바로 입장)
  - **링크 복사** / **공유하기(Web Share API, 모바일)**
- 라우팅: `/join/:code` 경로 추가 → 로그인 후 해당 코드로 자동 입장 플로우
- 라이브러리: `qrcode`(경량, 클라이언트 생성, 키 불필요)

### 기술 / DB
- DB 변경 없음
- 신규 의존성: `qrcode`
- 라우팅: `App.jsx`에 `/join/:code` 처리

### 영향 파일
- `src/pages/LobbyPage.jsx`(초대 시트), `App.jsx`(라우팅), 상단바 "복사" 버튼 확장

### 난이도: 하~중

## 7. 🗺️ 여행 로그 — 이미지 카드 공유 + 연도별 타임라인

### 7-1. 로그 이미지 카드 공유
- 완료 여행 상세를 **예쁜 카드 이미지(PNG)로 렌더 → 저장/SNS 공유**
- 카드 내용: 여행 이름·기간·목적지·대표 사진·한줄 후기
- 라이브러리: `html-to-image`(DOM → PNG) + Web Share API(`navigator.share`, 모바일)

### 7-2. 연도별 타임라인
- 여행 로그를 지도뿐 아니라 **연도별 타임라인**으로도 보기(2026 · 2025 …)
- 각 연도 아래 완료 여행 카드 나열, 정렬은 완료일 기준
- 지도 뷰(세계/한국)와 **탭 전환**(지도 · 타임라인)

### 기술 / DB
- DB 변경 없음(기존 `trips` 완료 데이터 사용)
- 신규 의존성: `html-to-image`

### 영향 파일
- `src/pages/TravelLogPage.jsx`(타임라인 탭), `TravelLogDetailPage.jsx`(공유 버튼), 관련 CSS

### 난이도: 중

## 8. 진행 전 결정 사항

### (A) 일정 지도 라이브러리 — **택1 필요** 🔴
| 옵션 | 장점 | 단점 |
|------|------|------|
| **Kakao Map API** | 한국 지도/장소검색 최강, 국내 여행에 최적 | API 키 발급 + 도메인 등록 필요 |
| **Leaflet + OpenStreetMap** | 키 불필요, 전 세계 커버, 의존성 가벼움 | 국내 장소검색·상세도는 카카오보다 약함 |

> 이 앱은 국내 여행 중심이지만 여행 로그는 해외도 포함 → **국내 위주면 Kakao, 해외까지 신경 쓰면 Leaflet** 추천.
> 결정 주시면 그에 맞춰 (3-2) 구현합니다. (드래그 정렬(3-1)은 지도와 무관하게 먼저 가능)

### (B) 신규 의존성 요약
- `qrcode` (로비 QR)
- `html-to-image` (여행 로그 공유)
- 지도 라이브러리 (Kakao SDK 또는 `leaflet`)

### (C) DB 마이그레이션 요약
- `excluded_regions` 테이블(홈) — 사용/생성
- `plans` 테이블에 `sort_order`, `lat`, `lng` 컬럼 추가(일정)
